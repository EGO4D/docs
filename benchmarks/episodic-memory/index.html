Episodic Memory

Benchmark Repo: https://github.com/EGO4D-Consortium/episodic-memory

Motivation‚Äã

Egocentric video from a wearable camera records the who/what/when/where of an individual‚Äôs daily life experience. This makes it ideal for what Tulving called episodic memory: specific first-person experiences (‚Äúwhat did I eat and who did I sit by on my first flight to France?‚Äù), to be distinguished from semantic memory (‚Äúwhat‚Äôs the capital of France?‚Äù). An augmented reality assistant that processes the egocentric video stream could give us super-human memory if it could appropriately index our visual experience and answer queries.

Task Definition‚Äã

Given an egocentric video and a query, the Ego4D Episodic Memory task requires localizing where the answer can be seen within the user‚Äôs past video. We consider three query types:

  1. Natural language queries (NLQ), in which the query is expressed in text (e.g., ‚ÄúWhat did I put in the drawer?‚Äù), and the output response is the temporal window where the answer is visible or deducible.
  2. Visual queries (VQ), in which the query is a static image of an object, and the output response localizes the object the last time it was seen in the video, both temporally and spatially. The spatial response is a 2D bounding box on the object, and optionally a 3D displacement vector from the current camera position to the object‚Äôs 3D bounding box. VQ captures how a user might teach the system an object with an image example, then later ask for its location (‚ÄúWhere is this [picture of my keys]?‚Äù).
  3. Moments queries (MQ), in which the query is the name of a high-level activity or ‚Äúmoment‚Äù, and the response consists of all temporal windows where the activity occurs (e.g., ‚ÄúWhen did I read to my children?‚Äù).

Annotations‚Äã

For language queries, we devised a set of 13 template questions meant to span things a user might ask to augment their memory, such as ‚Äúwhat is the state of object X?‚Äù, e.g., ‚Äúdid I leave the window open?‚Äù. Annotators express the queries in free-form natural language, and also provide the slot filling (e.g., X = window).

For moments, we established a taxonomy of 110 activities in a data-driven, semi-automatic manner by mining the narration summaries. Moments capture high-level activities in the camera wearer‚Äôs day, e.g., setting the table is a moment, whereas pick up is an action in our Forecasting benchmark.

For NLQ and VQ, we ask annotators to generate language/visual queries and couple them with the ‚Äúresponsetrack‚Äù in the video. For MQ, we provide the taxonomy of labels and ask annotators to label clips with each and every temporal segment containing a moment instance. In total, we have ‚àº74K total queries spanning 800 hours of video.
